Top 3 prioriteiten
De hoogste impact acties om nu aan te beginnen
1
Voeg JSON-LD schema markup toe (MobileApplication + FAQPage + Organization)
Geen enkele pagina heeft gestructureerde data. Voor de homepage: MobileApplication schema met naam, beschrijving, OS (iOS/Android), downloadUrl en rating. FAQPage schema voor de FAQ-sectie. Organization voor het merk. Dit is de enige manier om rich results (app-paneel, FAQ-snippets) in Google te krijgen.
Verwacht effect: rich results in Google, hogere CTR, adresseert "no rich results" issue direct.
Hoog
2
Voeg canonical tags toe aan alle 6 pagina's
Geen enkele pagina heeft een <link rel="canonical"> tag. Zonder canonical kan Google besluiten welke URL-variant te indexeren — met of zonder trailing slash, met UTM parameters, etc. Dit kan de "pages not indexed" problematiek verklaren. Voeg toe via Angular's Meta service of DOCUMENT injection in elke page component.
Verwacht effect: duidelijker indexeringsignaal voor Google, helpt pages-not-indexed probleem oplossen.
Hoog
3
Voeg OG + Twitter Card tags toe aan de overige 5 pagina's
Alleen de homepage heeft OG tags. /contact, /privacy, /terms, /remove-account en /wat-is-een-anytimer tonen een kale link wanneer gedeeld via WhatsApp, Instagram of iMessage. Voeg minimaal og:title, og:description, og:image en og:url toe via de Meta service in elk page component.
Verwacht effect: aantrekkelijke link previews op alle gedeelde pagina's, hogere klikratio op social.

Site-brede issues
Problemen die alle of meerdere pagina's raken
Canonical tags — volledig afwezig

0 van 5 indexeerbare pagina's hebben een canonical tag

Zonder canonical kan Google zelf beslissen welke URL als "echte" versie wordt gezien. Dit is een waarschijnlijke bijdrage aan de "pages not indexed" klacht. URL-variaties als https://anytimer.app/?utm_source=... of https://anytimer.app/# worden dan als aparte pagina's gezien.

Implementeer canonical tags centraal in AppComponent, vergelijkbaar met de hreflang implementatie. Stel per route de canonical in via DOCUMENT injection: <link rel="canonical" href="https://anytimer.app/[route]">.
Hoog
Schema markup — 0% dekking

Geen enkele pagina heeft JSON-LD gestructureerde data

Dit is de directe oorzaak van "geen rich results". Google kan de app niet herkennen als MobileApplication, kan geen FAQ-snippets tonen, en geen Knowledge Panel opbouwen voor het merk.

Prioriteit 1: Homepage krijgt MobileApplication + FAQPage + Organization. Prioriteit 2: /wat-is-een-anytimer krijgt FAQPage. Prioriteit 3: overige pagina's krijgen WebPage/ContactPage.
Hoog
Open Graph & Twitter Cards — 5 van 6 pagina's missen OG tags

OG tags alleen op homepage

/contact, /privacy, /terms, /remove-account en /wat-is-een-anytimer tonen een kale link bij delen via social media of messaging apps.

Voeg minimale OG set toe aan elke page component: og:title, og:description, og:image (logo.png voor legal pagina's, Together.png voor /wat-is-een-anytimer en /contact), og:url, og:type "website".
Hoog
Hreflang — x-default ontbreekt

Alleen hreflang="nl" ingesteld, geen x-default

De huidige implementatie in AppComponent zet hreflang="nl" correct per pagina. Echter, Google adviseert ook hreflang="x-default" toe te voegen voor bezoekers zonder taalvoorkeur (bv. via internationale links). Momenteel is de NL versie ook de enige versie.

Voeg naast hreflang="nl" ook hreflang="x-default" toe met dezelfde URL. Dit is een kleine toevoeging in de setHreflang() methode van AppComponent.

Security headers

Security headers onbekend / waarschijnlijk niet geconfigureerd

HSTS, Content-Security-Policy, X-Frame-Options, X-Content-Type-Options en Referrer-Policy zijn op de meeste Angular hosting platforms niet standaard ingesteld. Dit heeft beperkte directe SEO-impact maar Google kan HTTPS-kwaliteit meewegen als ranking signal.

Configureer HTTP response headers via de hosting provider (Firebase Hosting: firebase.json headers, Netlify: _headers file, Vercel: vercel.json headers).

Performance indicatoren
Resource-optimalisatie en laadpatronen

Afbeeldingen in PNG-formaat (geen WebP)

Together.png, Together2.png en de mockup-afbeeldingen zijn PNG. WebP is typisch 25-35% kleiner bij vergelijkbare kwaliteit, wat direct invloed heeft op de LCP (Largest Contentful Paint) score.

Converteer Together.png en Together2.png naar WebP. Gebruik <picture> met <source type="image/webp"> als primaire bron en de PNG als fallback. Tools: cwebp, Squoosh, of een Angular build plugin.
Medium

Geen <link rel="preload"> voor LCP-afbeelding

De hero-achtergrond Together.png is waarschijnlijk de LCP (grootste zichtbare content above the fold), maar er is geen preload-hint in index.html. De browser ontdekt de afbeelding pas nadat HTML en CSS zijn geparsed.

Voeg toe in index.html <head>: <link rel="preload" as="image" href="assets/Together.png" media="(min-width: 769px)"> en een tweede voor Together2.png op mobiel.
Medium

Lazy loading correct geïmplementeerd
